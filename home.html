Welcome to My Computer Science Web Site. A project developed by Adam Hennefer while enrolled at California State University East Bay. I chose to make this website because it is simple, functional and adaptable. It exhibits many common website features. These features are briefly summarize within the accordian presented below. This project represents my initial introduction into web developement.


Website Features - Technical Jargon:

  1. This sites uses HTML5 code and utilizes semantic tags (e.g., header, section, aside, footer, etc.).

  2. Styles are generally defined in separate CSS files which use each of the following selectors: element, id, class, and pseudo-class.

  3. The horizontal navigation bar with links allow the user to traverse the web pages of this site.

  4. The navigation bar persists at the top of the page when scrolling.

  5. The accordian feature you are using now gets functionality from JavaScript.

  6. Each page has a unique title that is represented in the tab of the browser.

  7. Appropriate meta data is provided in the HMTL code to help identify the page content.

The History page features a slideshow.

  1. The images are gathered in an HTML container and use a decent amount of CSS to make them fade during transitions.

  2. The functionality to change images comes from JavaScript.

  3. The images were compiled just for demonstation purposes.

The About page features an embedded video.

  1. One of my favorite features of HTML5 is the ability to easily embed video based media without the need for a video player plug-in.

  2. Video files can be directly embedded in a video element similiar to the way a picture is embedded, so long as the video is formatted as an mp4, webm or ogg file.

  3. The HTML5 video element comes loaded with various attributes which provide additional JavaScript functionality to manipulate playback control.

  4. The HTML5 video element also allows for Encrypted Media Extensions which are JavaScript API's for protected content. More information about EME can be found here https://www.w3.org/TR/encrypted-media/

The partners page features a thank you message.

  1. This page runs a script when the gif is double clicked.

  2. The script utilizes AJAX (via an XMLHttpRequest object) to access text that is stored in a file.

  3. The plain text file is then printed to the web page one character at a time to mimic a typewriter.

  4. The script sends its output text to an innerHTML content dividing element which is empty.

  5. This appears to split the page as a message is typed.

  1. The Members page & the Restricted page will redirect users to the login page if they are not signed in.

  2. The Login page and new_users_login page use PHP for the server-side scripting.

  3. The PHP scripts connect to an SQL database to authenticate or create a user account.

  4. If authenticated, new users recieve email confirmation with their username and password.

  5. Email is provided by reconfiguring the XAMPP sendmail function to work with Gmail's SMTP settings.

  6. Once signed in, a session is created which will allow access to the restricted pages.

  7. The Logout page deletes the user session which removes access to the restricted pages.

The objective of the fourth mini-project is to add session content as well as a user database to create a secure user-specific experience. You may add this content to your existing pages or add new pages to your website. At a minimum, your project should meet the following requirements:

  1. One of your pages must include a form consisting of at least three input element types. Associated with your form should be a script that reads in user input and produces output that is displayed somewhere on your webpage. As an example, if you were building a pizza ordering website, you could have a page with a form that lets users specify options for pizza size, sauce, toppings, etc. in such a way that, whenever any setting is changed, a script runs to calculate the total price of the pizza, which is then displayed.

  2. One of your pages should include a form whose action attribute is set to a PHP file on your website that you have developed. It is up to you to decide whether to use the GET or POST method for sending inputs to the server.

  3. Your PHP script should include a feature in which user input is logged into CSV (or similar) text files that are stored on the web server. To prevent these log files from growing too large, a separate log file should be created for each date with the filename “log_[4 digit year]_[2 digit month]_[2 digit day].csv.” As an example, if the user submits a form an Aug. 2, 2016, then the user input should be appended to a file called log_2016_08_02.csv. You will need to use the PHP functions fopen, fprintf, and fclose, which are very similar to their C counterparts. You will also need to use the various date related functions we covered in class.

  4. Your PHP script should include input validation such that if a user inputs an incorrect or empty value for a field (e.g., inputs text where a number is expected), the form is presented again to the user along with an error message. Invalid inputs should still be recorded into the log file.

  5. Assuming input validation succeeds, your PHP script should perform some sort of calculation with the input data and present results on either the same page or another page using HTML that is dynamically generated via PHP.

  6. Your PHP script should use strings, numerical types, associative arrays, and at least one function.

  7. Your website should link to a MySQL database of user information. Your database should consist of at least one table that contains columns for storing usernames, passwords, user email, user address, and any other relevant user information.

  8. You should have a registration page that allows new users to register (and have their information added to the database).

  9. Your website should have at least two restricted pages that cannot be accessed unless a user has logged in (see below requirement). If a user tries to access one of these pages without being logged in, then the user should be redirected to the login page.

  10. You should have a login page that lets a user provide his/her username and password. If the credentials match a row in your database table, then let the user access some restricted page in your site and create a session for that user. If the credentials do not match a row in your database table, then redisplay the login page, but with an error message.

  11. You should have a logout functionality in which the session for the user is terminated.

  12. You should create a predefined user with username = ‘admin’ and password = ‘hello’ for your and my testing purposes.
